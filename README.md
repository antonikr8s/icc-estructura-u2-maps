## Definici√≥n de Maps en Java

Un **Map** en Java es una estructura de datos que almacena pares clave-valor, donde cada clave es √∫nica dentro del mapa. Java proporciona varias implementaciones de `Map`, cada una con sus propias caracter√≠sticas:

3. **`TreeMap`**: Basado en un √°rbol rojo-negro, mantiene los elementos ordenados seg√∫n la clave.

### Comparaci√≥n entre `HashMap`, `LinkedHashMap` y `TreeMap`

| Caracter√≠stica      | `HashMap`          | `LinkedHashMap`      | `TreeMap`           |
|----------------------|-------------------|---------------------|---------------------|
| **Orden**           | No garantizado     | Orden de inserci√≥n  | Ordenado por clave seg√∫n `Comparable` o `Comparator` |
| **Duplicados (Claves)** | No permite         | No permite         | No permite         |
| **Eficiencia (B√∫squeda, Inserci√≥n, Eliminaci√≥n)** | O(1) promedio, O(n) peor caso | O(1) promedio, O(n) peor caso | O(log n) |

| **Necesita `equals()` y `hashCode()` en clave** | S√≠                 | S√≠                 | No, usa `compareTo()` o `Comparator` en la clave |
| **Uso recomendado** | Cuando se necesita rendimiento sin importar el orden | Cuando se necesita rendimiento manteniendo el orden de inserci√≥n | Cuando se necesita mantener los elementos ordenados por clave |

**Nota:** Si el `TreeMap` almacena objetos personalizados como clave, se recomienda utilizar un **`Comparator`** para definir el criterio de ordenaci√≥n en lugar de depender del orden natural del objeto.

### Consideraciones sobre `hashCode()` y `equals()`

Un `TreeMap` **no usa** `hashCode()` ni `equals()` para determinar la unicidad de claves. En su lugar, usa el `compareTo()` de `Comparable` o el `Comparator` definido.

Si se usara un `HashMap` en lugar de `TreeMap`, entonces `hashCode()` y `equals()` en la clave ser√≠an necesarios para evitar colisiones.
